## ✨ What Is This?

VoiceIQ is a **sub-100 MB, fully client-side AI** that enhances automated voice interviews directly in the browser. No API keys. No cloud. No network latency.

### The Problem It Solves

Automated voice interviews feel robotic when there are silences. VoiceIQ fills those pauses with **AI-generated filler phrases that directly reference what the candidate just said** — while simultaneously building a **live summary** of the conversation.

### Key Features

| Feature | Detail |
|---|---|
| 🎯 **Conversation-aware fillers** | Extracts real phrases from what was just said and mirrors them back |
| 🧠 **Real AI inference** | `flan-t5-small` via ONNX Runtime — fully in the browser |
| ⚡ **Sub-millisecond latency** | Mirror engine is instant; AI pre-generates the next filler async |
| 🎤 **Live transcription** | Web Speech Recognition with interim results |
| 📝 **Real-time summaries** | Rolling summary updated after each pause |
| 🔇 **Pause detection** | Web Audio API — triggers after 2.2 seconds of silence |
| 🔊 **TTS playback** | Web Speech Synthesis speaks every filler aloud |
| 📴 **Fully offline** | After first load, works with no internet |
| 📏 **~66 MB total** | Well under the 100 MB constraint |

## 🚀 Quick Start

### Prerequisites

- **Python 3.8+** — [Download](https://python.org/downloads)
- **Chrome or Edge** — required for Web Speech API
- Internet connection for **first run only** (downloads model ~66 MB)

---

### Step 1 — Clone

```bash
git clone https://github.com/Zeeshan2912/AI-Interview-Assistant.git
cd voiceiq
```

### Step 2 — Install Dependencies

```bash
pip install -r requirements.txt
```

### Step 3 — Build

```bash
python build.py
```

Downloads `Xenova/flan-t5-small` and generates `dist/index.html`.

```
INFO - ✅ onnx/encoder_model_quantized.onnx  (24.8 MB)
INFO - ✅ onnx/decoder_model_merged_quantized.onnx  (40.1 MB)
INFO - Model directory: 67.3 MB
INFO - ✅ Build complete!
```

> Download happens once. Subsequent builds reuse cached files.

### Step 4 — Serve

```bash
cd dist
python -m http.server 8000
```

> ❗ Must use HTTP server — not `file://`. Keep this terminal open the whole time.

### Step 5 — Open

```
http://localhost:8000
```

Wait 20–40 seconds for:
```
✅ Model loaded in XXXX ms — offline ready
```

## 🎮 Using the Demo

### Quick Filler Test (No Mic Needed)

1. Wait for green badge
2. Click **✏️ Add text** → type: `I built a REST API using Python that handled 10000 requests per second`
3. Click **✏️ Add text** again → type: `I led a team of 8 engineers through the migration to AWS`

## 🎯 Filler System Deep Dive

### The Core Idea: Mirror + React

Instead of pre-written generic phrases, VoiceIQ extracts key information from what was just said and reflects it back directly.

**❌ Generic (bad):** *"Interesting."* — said after anything, completely disconnected.

**✅ Mirror (good):** *"Built with Python — that takes real skill."* — directly references what was said.

### Real Conversation Examples

| Candidate says | Extracted phrase | Filler generated |
|---|---|---|
| *"I reduced API latency by 40% using Redis"* | `reduced API latency by 40%` | **"That reduced API latency by 40% — really impressive."** |
| *"Built the whole system with React and Node"* | `built with React` | **"Built with React — that takes real skill."** |
| *"Led a team of 8 engineers through migration"* | `team of 8 engineers` | **"Getting team of 8 engineers right is no small feat."** |
| *"We failed to ship on deadline due to a bug"* | *(challenge detected)* | **"That sounds like a genuinely tough situation."** |
| *"We hit 2x revenue growth after the launch"* | *(achievement detected)* | **"Results like that don't come without serious effort."** |
| *"Can you tell me about the team structure?"* | *(question detected)* | **"That is a great question — give me a moment."** |

### Key Phrase Extraction — Priority Order

The extractor scans the last sentence for these patterns:

| Priority | Pattern | Example |
|---|---|---|
| 1st | Action + Number | "reduced latency by 40%" |
| 2nd | Action + Tech | "built with React" |
| 3rd | Team size | "team of 8 engineers" |
| 4th | Pure number/metric | "10,000 requests per second" |
| 5th | Tech name | "Kubernetes" |
| 6th | Company name | "working at Google" |
| 7th | Action phrase | "led the migration" |

### Three-Tier Filler Architecture

```
2.2s silence detected
       │
       ▼
┌──────────────────────┐
│  Tier 1: nextFiller  │  Pre-computed on previous pause — served instantly
└──────────┬───────────┘
           │ (if empty)
           ▼
┌──────────────────────┐
│  Tier 2: Mirror      │  Extracts key phrase + slots into template — <1ms
└──────────┬───────────┘
           │ (if no entities)
           ▼
┌──────────────────────┐
│  Tier 3: Pool        │  Context-matched static phrase — <1ms
└──────────────────────┘
           │
           ▼
    Spoken via TTS + shown on screen

    [async background]
┌──────────────────────┐
│  AI Prefetch         │  flan-t5 generates next filler (~500ms)
│                      │  Stored as nextFiller for next pause
└──────────────────────┘
```

### Entity Recognition — Zero Regex

All entity extraction uses **pure string methods** (`.indexOf()`, `.split()`) — no regular expressions. This avoids Python→HTML escaping bugs entirely.

## 🧠 Architecture

```
┌────────────────────────────────────────────────────┐
│                    Browser                         │
│                                                    │
│  Microphone ──► Web Audio API ──► Pause Detector   │
│                 (512-sample FFT)   (2.2s threshold) │
│                                         │          │
│  Web Speech ──► Transcript         Filler Engine   │
│  Recognition    (live text)         (instant <1ms) │
│                     │                   │          │
│                     ▼                   ▼          │
│              Transformers.js    Web Speech Synthesis│
│              flan-t5-small      (speaks filler)    │
│              ONNX Runtime                          │
│              (async prefetch)                      │
└────────────────────────────────────────────────────┘
```

### Why `flan-t5-small`?

| Model | Size | Verdict |
|---|---|---|
| `tiny-random-llama` | 15 MB | ❌ Random weights — outputs gibberish |
| `distilgpt2` | 237 MB | ❌ Exceeds 100 MB limit |
| `gpt2` | 326 MB | ❌ Exceeds 100 MB limit |
| **`flan-t5-small`** | **66 MB** | ✅ Instruction-tuned, follows prompts correctly |

### Why `allowRemoteModels = true`?

Transformers.js v2 treats any `http://` URL — including `http://localhost` — as **remote**. The correct configuration:

```javascript
env.localModelPath    = window.location.origin + '/';
env.allowLocalModels  = true;
env.allowRemoteModels = true;  // localhost is classified as remote
pipe = await pipeline('text2text-generation', 'model', { quantized: true });
// Resolves to: http://localhost:8000/model/config.json ✅
```

## 🐛 Troubleshooting

### ❌ `allowRemoteModels=false` error in Console
```powershell
python -c "f=open('index.html',encoding='utf-8');c=f.read();f.close();c=c.replace('allowRemoteModels = false','allowRemoteModels = true');open('index.html','w',encoding='utf-8').write(c);print('Done')"
```

### ❌ 404 on model files
Model wasn't downloaded. Run from project root:
```bash
python build.py
```

### ❌ `ERR_CONNECTION_REFUSED`
Server stopped. Restart it and keep terminal open:
```bash
cd dist && python -m http.server 8000
```

### ❌ `Invalid regular expression` in Console
Old version of `index.html`. Download and replace with latest from repo.

### ❌ Port 8000 in use
```bash
python -m http.server 8080
# Open http://localhost:8080
```

### ❌ No sound
- Check system volume
- Click mic button first (browsers need a user gesture before audio)

### ❌ Microphone not working
- Use Chrome or Edge only
- Click Allow when prompted
- Use ✏️ Add text as fallback

### ❌ Server shows directory listing
Started from wrong folder:
```bash
cd dist   # must be inside dist/
python -m http.server 8000
```
